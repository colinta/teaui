# Core API (OOP)

The core API lets you build UIs by directly instantiating `View` objects. This is the foundation that the React renderer builds on.

## Hello World

```js
import {
  Screen,
  Box,
  Stack,
  Text,
  Button,
  interceptConsoleLog,
} from '@teaui/core'

interceptConsoleLog()

Screen.start(
  new Box({
    border: 'single',
    children: [
      Stack.down([
        new Text({text: 'Why is it called TeaUI?'}),
        new Button({title: 'Tell me!'}),
      ]),
    ],
  }),
)
```

## Key Concepts

### Views

Every UI element is a `View`. Views have:

- **`naturalSize(available)`** — reports preferred size given available space
- **`render(viewport)`** — draws itself into a `Viewport`
- Props like `width`, `height`, `flex` for layout

### Containers

[`Container`](/docs/components/container) extends `View` to hold children. `Box`, `Stack`, `Window`, `Accordion`, `Tabs`, and `Drawer` are containers. Use `container.add(child)`, `container.removeChild(child)`, and `view.removeFromParent()` to manage children imperatively.

### Screen

`Screen.start(rootView)` initializes the terminal and starts the render loop:

```js
import {Screen, Text} from '@teaui/core'

Screen.start(new Text({text: 'Hello!'}))
```

### Updating Views

Call `view.update(newProps)` to change a view's properties. This triggers a re-render.

```js
const text = new Text({text: 'Hello'})
// Later:
text.update({text: 'Updated!'})
```

### Events

Interactive views support:

- **`onClick`** — click/enter handler (Button)
- **`onChange`** — value change handler (Input, Checkbox, Slider)
- **`onSubmit`** — submit handler (Input)

Mouse events are automatic for views that render in the viewport.

## When to Use Core API

- **Simple scripts** — quick terminal UIs without React
- **Extending TeaUI** — building custom views
- **Understanding internals** — the React API maps directly to these views

For most applications, the [React API](/docs/getting-started) is recommended.
